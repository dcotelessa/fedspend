## Notes
Behavior-preserving view-layer polish (see Angular deep-arch review). Three independent fixes, each with an existing in-repo pattern to copy:

(1) STABLE CHART-INPUT REFERENCES. ng2-charts BaseChartDirective re-renders whenever its [data]/[datasets]/[options] input REFERENCE changes. Inline literals in the template create a NEW object/array on every change-detection cycle, so the chart redraws on every CD (HTTP completion, tab clicks, selects). Offenders: disaster-lens.component.html binds [datasets]="[{ label: 'Obligated Amount', data: top15Datasets() }]" (new array each CD); dashboard.component.html binds [data]="{ labels: chartLabels(), datasets: chartDatasets() }" and [options]="{...}" (new objects each CD). FIX: hoist to computed/readonly fields in the component. COPY THE EXISTING PATTERN: agency-spotlight.component.ts already does 'readonly chartData = computed<ChartData>(() => ...)' and binds [datasets]="chartData().datasets"; geographic-view.component.ts binds [datasets]="chartData()" (a computed). For disaster-lens add 'readonly top15ChartDatasets = computed<ChartDataset[]>(() => [{ label: 'Obligated Amount', data: this.top15Datasets() }])' and bind it. For dashboard add 'readonly chartData = computed(() => ({ labels: this.chartLabels(), datasets: this.chartDatasets() }))' and a 'readonly chartOptions = { responsive: true, maintainAspectRatio: false, plugins: { legend: { display: false } } }' field (options are static so a plain readonly field is fine — no need for computed), then bind [data]="chartData()" [options]="chartOptions".

(2) GEOGRAPHIC-VIEW PAGINATOR. geographic-view.component.html has <mat-paginator [length]="paginator.length" ...> with NO (page) handler, and the table binds [dataSource]="allStates()" (all rows) — so the paginator is dead UI. FIX: wire paging. COPY disaster-lens: add pageIndex/pageSize signals + an onPage handler + a pagedStates computed (sortedStates().slice(start, start+pageSize)), bind the table [dataSource]="pagedStates()" and the paginator [length]="allStates().length" (page)="onPage($event)". Update the spec to assert paging state.

(3) MODERN CONTROL FLOW. disaster-lens.component.html and geographic-view.component.html still use legacy *ngFor/*ngIf; agency-spotlight already uses @for/@if. Convert *ngFor -> @for (with track) and *ngIf -> @if for consistency. Same rendered output, so specs that query DOM are unaffected.

Do NOT touch ApiService, interceptors, theme/loading services, or any backend. The chart re-render fix is the highest-value item (real CPU saving on interaction); the other two are consistency. Tests are table-driven (AGENTS.md Skill 2); the existing chart/paginator specs need only minimal updates (the dashboard chart spec already reads component.chartLabels()/chartDatasets() — keep those computeds, just add the wrapping chartData).
