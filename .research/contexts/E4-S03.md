## Notes
Single deep module per AGENTS.md Skill 3 — one ApiService, one method per use case, HttpClient injected, base URL from environment.apiUrl. provideHttpClient is NOT wired in this story (deferred to E4-S04); tests use provideHttpClientTesting + HttpTestingController, so do NOT edit app.config.ts. CRITICAL RESPONSE SHAPES — the backend has NO @Paginated decorator, so its ResponseWrapperInterceptor passes everything through RAW. ONLY GET /agencies returns a wrapped {data,meta} object (the service pre-wraps it); every other endpoint returns a raw array or raw object. So ONLY getAgencies maps r.data; the other 8 methods return the body directly (r). Per-method contract: getAgencies() GET /agencies -> {data,meta}; map r.data; the item shape is {id,name,totalCents} (NOT the shared Agency shape — backend drift), so define a local AgencyWithTotal interface. getAgencySummary(id) GET /agencies/:id/summary -> raw AgencySummary | null; return r. getAgencySpotlight(id) GET /agencies/:id/spotlight -> raw SpendingRecord[] | null; return r. getGeographyStates(params) GET /geography/states with HttpParams {fiscalYear?,agencyId?,scope?} -> raw GeoSpendingSnapshot[]; return r. getGeographyState(code) GET /geography/state/:code -> raw GeoSpendingSnapshot[]; return r. getDisasterOverview() GET /disaster/overview -> raw ARRAY DisasterOverview[] (note: array, not a single object); return r. getDisasterStates(params) GET /disaster/states with HttpParams {defGroup?,fiscalYear?} -> raw DisasterFundingRecord[]; return r. getDisasterRecoveryRatios(params) GET /disaster/recovery-ratios with HttpParams {fiscalYear?} -> raw DisasterRecoveryRatio[]; return r. getDisasterState(code) GET /disaster/state/:code -> raw state-profile object (no matching @shared/interfaces type — define a local DisasterStateProfile interface). Import matching types from @shared/interfaces (GeoSpendingSnapshot, DisasterRecoveryRatio, SpendingRecord, AgencySummary, DisasterOverview, ApiResponse). Build HttpParams skipping undefined values. catchError fallbacks: of([] as T[]) for list methods, of(null) for single/null methods. Spec (api.service.spec.ts) TABLE-DRIVEN: one testTable row per method {name, methodName, args, expectedUrl, mockBody, expected}; TestBed with provideHttpClientTesting, inject HttpTestingController; for each row call the method, expectOne(expectedUrl), flush mockBody (the REAL backend shape per the contract above), assert the emitted value equals expected (post-map: r.data for getAgencies, the body for the rest). No comments in code.
