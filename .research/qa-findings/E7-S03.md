# QA Findings — E7-S03
## Flagged for a follow-up story
- Non-functional matSort on recovery-ratio table: the `<table mat-table ... matSort>`
  and `matSortHeader` on the Ratio column are decorative. `dataSource` is the plain
  array `pagedRatios` (a sliced view of `sortedRatios`), not a `MatTableDataSource`,
  and there is no `@ViewChild(MatSort)` wiring, so clicking the sort arrow toggles
  the indicator but never re-sorts the rows. The intended behavior (per story notes)
  is a fixed client-side ascending sort, which works. Properly enabling user-toggleable
  sorting requires converting `sortedRatios`/`pagedRatios` to a `MatTableDataSource`
  with `.sort`/`.paginator` assigned in `ngAfterViewInit` — that is a behavior change
  (new user-facing capability + a real `MatSort` ViewChild + reintroducing the
  `AfterViewInit` lifecycle) and so belongs in a dedicated story rather than this
  behavior-preserving QA pass. Until then the dead `MatSortModule` import,
  `matSort`, and `matSortHeader` remain so the visible header is unchanged.
