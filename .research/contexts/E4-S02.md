## Notes
Angular 20 standalone components. CRITICAL — the scaffold ALREADY wires routing: app.config.ts has providers: [provideBrowserGlobalErrorListeners(), provideZoneChangeDetection({eventCoalescing:true}), provideRouter(routes)] importing `routes` from './app.routes', and main.ts does bootstrapApplication(App, appConfig). So: DO NOT create a separate routes.ts; DO NOT add a second provideRouter anywhere (double-provisioning breaks the router at runtime); DO NOT edit main.ts or app.config.ts. Your ONLY routing change is populating frontend/src/app/app.routes.ts (replace `export const routes: Routes = [];`). Tasks: (1) Create 5 standalone placeholder components, each in <name>.component.ts exporting class <Name>Component — DashboardComponent (dashboard), GeographicViewComponent (geographic-view), AgencyListComponent (agency-list), AgencySpotlightComponent (agency-spotlight), DisasterLensComponent (disaster-lens). Components are standalone BY DEFAULT in Angular 20 (omit the `standalone` flag). Each @Component: selector 'app-<name>', templateUrl '<name>.component.html', and the .component.html contains a single <h1>heading</h1>. No imports needed for placeholders. (2) In app.routes.ts define: { path: '', component: DashboardComponent }, { path: 'geography', component: GeographicViewComponent }, { path: 'agencies', component: AgencyListComponent }, { path: 'agencies/:id', component: AgencySpotlightComponent }, { path: 'disaster', component: DisasterLensComponent } — import each component, use single-quoted path strings, eager `component:` mapping. (3) routes.spec.ts: TABLE-DRIVEN — import `routes` from './app.routes' and all 5 component classes; build a testTable of {path, expectedComponent} rows; for each row, find the route in `routes` with that path and assert route.component === expectedComponent. No RouterTestingModule/harness needed — assert directly on the routes array. The root component class is `App` (Angular 20 renamed it from AppComponent), referenced only by main.ts — leave it alone.
