# QA Findings — E6-S02

## Flagged for a follow-up story

- **FY range selector dropdowns are non-functional**: `availableYears` is
  declared (`= []`) but never populated, so both `<select>` elements render with
  zero options — the user can never change the range via the UI (only the
  hardcoded 2020–2024 defaults are in effect). Fixing it changes observable
  behavior (the selects must be populated, e.g. from the union of fiscal years
  in the loaded records / a fixed range) and is therefore out of scope for a
  behavior-preserving QA pass. While wiring it, also fix the latent
  `[(ngModel)]` over `<option [value]="year">` coercion (the bound value becomes
  a string on selection, which would break the numeric comparisons in
  `buildStackedChartFromRecords`), and consider switching to `mat-select` as the
  story context specified ("two mat-selects from/to").

- **Agency header and badge are never populated**: `agency`, `badgeColor`, and
  `badgeText` are bound in the template but never assigned in `ngOnInit`, so the
  header (name / abbreviation / total) and badge always render empty. The data
  arrives via `ApiService.getAgencySummary(id)`, which exists but is unused here.
  Fixing requires a second fetch + badge-color logic — an observable-behavior
  change, so it belongs in a follow-up.

- **AWARD_COLORS is never applied to the chart**: the constant is correct and
  tested, but the component can't actually use it because `ChartDataset`
  (in `bar-chart.component.ts`, outside this story's closed set) only carries
  `label` + `data` with no color fields, and the bar-chart template doesn't read
  a color. Coloring the stacked segments requires extending that interface and
  template (a cross-file behavior change) — follow-up. (The dead `AWARD_COLORS`
  import in the component was removed during this QA pass.)

- **FY range is not sent to the API**: the story context states fetching goes via
  `getAgencySpotlight(id, {fiscalYearStart, fiscalYearEnd})`, but
  `ApiService.getAgencySpotlight(id)` accepts no query and filtering is done
  entirely client-side over the full record set. The `AgencySpotlightQuery`
  interface already exists in `shared/interfaces/index.ts` but is unused. Wiring
  server-side range filtering needs ApiService + backend changes. Low priority:
  the client-side filter already produces correct chart output.

## Changes applied in this QA pass (behavior-preserving)
- `agency-spotlight.component.ts`: removed unused `AWARD_COLORS` import and the
  unused `AwardTypeSum` interface; renamed the misleading `routeParam`
  (an `ActivatedRoute`) to `route`; rewrote `buildStackedChartFromRecords` as a
  single-pass transform (Map of year→sum per award type) instead of the previous
  O(awardTypes × years) repeated full-scan — same observable output, deeper
  module.
- `agency-spotlight.component.spec.ts`: removed dead `httpMock`/`HttpClient`;
  added an `expectedData` assertion (per-award-type summed `data` arrays) that
  sharpens every existing row without weakening prior assertions; added two
  edge-case rows — "fills missing award-type buckets within a year with zero"
  and "sorts years and award types regardless of input order".
