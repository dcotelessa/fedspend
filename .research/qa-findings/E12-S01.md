# QA Findings — E12-S01

## Flagged for a follow-up story
- Inconsistent pagination meta in `AgenciesService.findAllWithTotals`: the
  returned `meta` is `{ total: data.length, page: 1, pageSize: 6 }`. `total`
  reflects the real agency count (can be >100) while `page`/`pageSize` are
  hardcoded magic numbers with no corresponding limit/offset query and no
  pagination input on the method. A client reading `pageSize: 6` alongside a
  large `total` is misled about result set shape. Correcting the values (or
  removing the page/pageSize fields / adding real pagination params) is an
  observable API change and therefore out of scope for behavior-preserving QA.

## Resolved within QA (not a follow-up)
- `.select([...])` was aliasing `COALESCE(SUM(sr.obligatedAmount), 0)` and a
  bare `'totalCents'` as two array elements. TypeORM's array-select form does
  not alias aggregates, so the bare `'totalCents'` would select a phantom
  column and `row.totalCents` would be undefined against a real DB (hidden only
  by the mock). Refactored to `.select([...]).addSelect(expr, 'totalCents')`,
  which matches the story's documented intent. Public API and return shape
  unchanged.
