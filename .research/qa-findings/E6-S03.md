# QA Findings — E6-S03
## Flagged for a follow-up story
- Stale `currentRecords` on cross-agency re-navigation: `AgencySpotlightComponent`
  subscribes to `route.paramMap`, so navigating from `/agency/:a` to `/agency/:b`
  reuses the same component instance and re-emits. In the spotlight `next` handler
  `this.currentRecords = records` only runs inside `if (records && records.length > 0)`.
  When agency `b` returns `[]` or the request resolves to `null` (ApiService catches
  to `of(null)`), `currentRecords` keeps agency `a`'s records, so `computeInsight()`
  and `buildTableData()` render the previous agency's data. Fix is a behavior change
  (unconditionally reset to `records ?? []` and rebuild) and there is no spec row
  covering a second param emit, so it is out of scope for this behavior-preserving
  QA pass. Belongs in a story that owns the agency-spotlight navigation lifecycle.
