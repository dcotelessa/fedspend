## Notes
Angular 20. CRITICAL: providers go in app.config.ts, NOT main.ts. main.ts just does bootstrapApplication(App, appConfig) — do NOT edit it. In app.config.ts add to the EXISTING providers array: provideHttpClient(withInterceptors([loadingInterceptor, errorInterceptor])) AND provideAnimationsAsync() (import from @angular/platform-browser/animations/async — MatSnackBar needs animations at runtime; without it, opening a snack-bar throws). Keep the existing provideRouter(routes), provideBrowserGlobalErrorListeners(), and provideZoneChangeDetection. Interceptors are FUNCTIONAL (HttpInterceptorFn from @angular/common/http), NOT classes. LoadingService (@Injectable({providedIn:'root'})): a private counter field, increment()/decrement() methods, and readonly loading$ = signal(false) as a WritableSignal<boolean> (signal from @angular/core). decrement clamps counter at >=0; both methods call loading$.set(this.counter > 0). Only loading$ and increment/decrement are public. LoadingInterceptor (HttpInterceptorFn): inject(LoadingService), call increment() before forwarding, return next(req).pipe(finalize(() => loadingSvc.decrement())) so the counter decrements on BOTH success and error. ErrorInterceptor (HttpInterceptorFn): inject(MatSnackBar), return next(req).pipe(catchError((error: HttpErrorResponse) => { const msg = error.status ? error.status + ': ' + error.message : 'Network error'; snackBar.open(msg, 'Dismiss', { duration: 5000 }); return throwError(() => error); })) — RE-THROW after showing the snack-bar so the ApiService catchError fallbacks still fire (the service still returns [] / null to its subscribers); do NOT swallow. Specs: loading.service.spec.ts TABLE-DRIVEN over a testTable of {operations, expectedLoading} rows asserting loading$() after increment/decrement sequences. interceptors.spec.ts: TestBed with provideHttpClient(withInterceptors([loadingInterceptor, errorInterceptor])) + provideHttpClientTesting, HttpTestingController; for LoadingInterceptor assert the LoadingService counter/loading$ toggles around a flushed request; for ErrorInterceptor spy on MatSnackBar.open (inject MatSnackBar, jest.spyOn) and assert open() is called with the message AND the error re-throws (subscribe with error handler).
