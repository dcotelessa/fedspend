# QA Findings — E15-S01

## Flagged for a follow-up story
- Insight text goes stale on FY range change: `AgencySpotlightComponent.onRangeChange()`
  rebuilds the chart and the breakdown table but never re-runs `computeInsight()`.
  Because `computeInsight()` embeds the selected range in its output string
  (`"In FY{start}-{end}, {agency} spent {pct}% on {type}."`), narrowing the From/To
  dropdowns leaves the insight line showing the original full range while the rest
  of the view updates. Fix = add `this.insight = this.computeInsight();` to
  `onRangeChange()`, plus a table-driven spec row that changes the range and asserts
  the insight string updates. Not done in QA because it is an observable behavior
  change (the spec contract covers `computeInsight` only in isolation, never driven
  through `onRangeChange`), and QA is behavior-preserving.

## Notes (no action needed — documented for completeness)
- `populateAvailableYears()` with an empty record set yields
  `fiscalYearStart = Infinity` / `fiscalYearEnd = -Infinity` (from
  `Math.min/max(...[])`). This is not an active bug because `ngOnInit` guards the
  call behind `if (this.currentRecords.length > 0)`, and the behavior is now
  codified by the `availableYearsTestTable` empty-records row. A future story could
  make the method self-defensive if an unguarded caller is ever introduced.
