## Notes
The USASpending API has no /search/spending_by_agency/ endpoint (404). BUT /search/spending_by_geography/ accepts an agencies filter in the POST body: { filters: { time_period: [...], agencies: [{ type: 'awarding', tier: 'toptier', toptier_code: '097' }] }, geo_layer: 'state' }. This returns per-state spending for ONE agency.

Approach: for each agency in the agency table, POST to /search/spending_by_geography/ with that agency's toptier_code. Sum all states' aggregated_amount to get the agency total. Create one SpendingRecord per agency per fiscal year with awardTypeLabel='Total'.

Update syncAgenciesAndSpending to loop through agencies (limit to top 20 by spending to avoid 111 API calls). For each: fetch spending, sum amounts, create SpendingRecord.

Update fetchSpendingByAgency params to accept toptier_code instead of agency string. Remove the empty-agency guard (it was added because the old code passed empty string).
