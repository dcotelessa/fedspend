# QA Findings — E14-S07

## Flagged for a follow-up story

- [syncGeography delete scoping]: The spec verifies `geoRepo.delete` is called N times but does NOT assert the fiscalYear argument on each call. A broken implementation (e.g., deleting without fiscalYear scoping) could still pass. A future story should add `expectedDeleteScopes` to the TestCase interface and verify each delete was called with `{ fiscalYear: X, scope: 'recipient', agencyId: IsNull() }`.

- [syncAll year-specific data]: The syncAll test uses a single mock row returned for all 5 year fetches, not verifying that each year receives year-appropriate data. A future story should use year-specific mock data to ensure the loop processes each year independently.
