## Notes
The parens fix is GLOBAL (currencyFormat is used across Geography/Agencies/Disaster), but Geography and Agencies data currently has no negative obligations, so only Disaster 88 negative fedSpendingObligated rows visibly flip from ($x) to -$x. The negatives are real USASpending de-obligation/adjustment values - the minus sign preserves the information honestly rather than hiding it. The delete is safe because Intl.NumberFormat already produces -$12,847,333.64 for negatives; the manual replace block was overriding that to accounting parens. The Disaster component SCSS mirrors geographic-view.component.scss and agency-spotlight.component.scss conventions (M3 --mat-sys-* tokens, surface-container cards, level1 shadow, corner vars). No backend change; no pi/build-agent run required (harness=opencode is fine). Does NOT address the ratio=0 / FEMA-missing issue - that is E16-S02.
