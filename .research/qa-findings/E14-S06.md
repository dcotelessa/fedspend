# QA Findings — E14-S06
## Flagged for a follow-up story
- NaN in vsAvg when average obligatedAmount is zero: In `processData()`,
  `vsAvg` is computed as `((s.obligatedAmount - avg) / avg) * 100`. If every
  loaded `GeoSpendingSnapshot` has `obligatedAmount === 0`, then `avg === 0`
  and every `vsAvg` entry becomes `NaN`, rendering `NaN%` in the All States
  table. The correct semantic value is `0` (all states equal the zero average),
  so a guard such as `avg === 0 ? 0 : ((s.obligatedAmount - avg) / avg) * 100`
  would fix it. Not fixed in this QA pass because changing `NaN` to `0` is an
  observable behavior change, which the behavior-preserving QA contract
  forbids. Needs its own story with a dedicated testTable row (non-empty
  primary data where every row has `obligatedAmount: 0`, expecting `vsAvg: [0, ...]`).
